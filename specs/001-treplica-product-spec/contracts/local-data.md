# Contract: Local Data, Privacy, and Audit

> Export/import detalhado: [app-settings.md](./app-settings.md).  
> Mapa: [ecosystem.md](../ecosystem.md)

## Purpose

Define how the app stores, logs, exports, imports, and deletes user data while preserving the local-first promise.

## Local Storage Rules

- Sessions, transcripts, translations, guidance, generated documents, audit logs, provider metadata, and user settings are stored on the user's machine.
- **App settings** (hotkeys, onboarding, model routing, assistant defaults, export directory) live in `app_settings.json` beside `treplica.db`.
- Provider credentials are stored through OS-backed secret storage when available and referenced by identifier.
- No cloud sync, telemetry, analytics, or remote backup is enabled by default.
- Hosted provider requests are allowed only after user configuration and privacy approval.

## Export directory resolution

1. If `app_settings.documents_export_dir` is set → use that absolute path (created if missing).
2. Else → `{app_data_dir}/exports`.

Layout per session: `{exports_root}/{session_id}/{doc-type-slug}-{doc-id-prefix}.md`

Each file includes YAML front matter:

```yaml
---
title: ...
session_id: ...
doc_type: ...
exported_at: ...
---
```

## Import rules

- User selects a folder; app recursively scans `*.md`.
- Files MUST contain Treplica front matter with `session_id` (or infer from parent folder name).
- Existing `storage_path` in DB → skip (idempotent).
- Missing session → create **imported** session row (`status: ended`) + empty context.
- Import does **not** restore transcripts, translations, or guidance — documents only.

## Audit Log Rules

Audit log entries MUST be created for:

- Session started, paused, resumed, ended, failed, deleted.
- Audio capture permission changes.
- Provider configured, enabled, disabled, tested, or used.
- Hosted provider request sent.
- Generated document created, exported, copied, or deleted.
- **Documents imported** (batch summary when applicable).
- Privacy mode changed.
- **Export directory changed**.
- Local data deletion completed or failed.
- Critical app errors.

Audit logs MUST NOT include:

- Full API keys.
- Raw credentials.
- Sensitive full prompt payloads unless the user explicitly enables verbose local logs.
- Hidden copies of deleted session content after deletion completes.

## Deletion Rules

When a user deletes a session, the app MUST delete:

- Session row.
- Transcript and translation segments.
- Guidance suggestions.
- Generated documents and exported local files when `storage_path` points to an existing file.
- Session subdirectory under **effective** `exports_root` when present.
- Provider call raw payloads if stored.
- Search indexes.
- Session-specific caches.

Files the user copied manually outside app-managed paths are not tracked.

When a user deletes a single document, only that DB row and its `storage_path` file are removed.

## Export Rules

- Users may export transcripts, summaries, logs, and generated documents to local files.
- Exports must clearly show whether the content was generated by AI and which provider was used when available.
- Export must never include provider credentials.
- Re-export overwrites the same filename pattern for the document id.

## Contract Tests

- Deleting a session removes related rows and app-managed files.
- Custom export dir receives new documents; reset restores default in settings.
- Import from folder adds documents and skips duplicates.
- Hosted provider use creates an audit log entry with `local_or_hosted = hosted`.
- Credential references do not expose raw credentials in database exports.
- Verbose logs are off by default.
- Exported document includes provenance metadata.
